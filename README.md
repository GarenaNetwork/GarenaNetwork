## Overview
Garena Singapore uses this repository to run developer assessments, evaluate technical fluency, and prototype engineering ideas. The hub provides a controlled environment for candidates to demonstrate problem-solving skills while allowing reviewers to observe code quality, testing rigor, and collaboration habits.

## Focus Areas
### Game & Platform Development
- Core gameplay systems and tooling
- Live-ops features and infrastructure
- Player experience instrumentation

### Backend & Distributed Infrastructure
- High-throughput services and APIs
- Real-time data processing pipelines
- Reliability engineering and observability

### Emerging Technology & Innovation
- Web3 and digital asset experiments
- GameFi and social interaction layers
- AI-powered game and platform capabilities

### Developer Experience & Internal Tools
- Productivity tooling and automation
- CI/CD and deployment workflows
- Knowledge sharing and technical enablement

## Assessment Workflow
1. **Invitation** – Candidates receive scoped tasks aligned with their role.
2. **Execution** – Work is completed within this repository following provided guidelines.
3. **Review** – Engineers evaluate implementation quality, tests, and trade-off discussions.
4. **Feedback** – Outcomes inform subsequent technical conversations or offers.

## Candidate Expectations
- Follow repository conventions and existing tooling.
- Write maintainable, well-tested code without introducing secrets.
- Document assumptions within pull requests or code where appropriate.
- Communicate blockers early and propose pragmatic solutions.

## Engineering Principles
- **Clarity** – Favor simple, explicit designs over clever shortcuts.
- **Precision** – Validate changes through tests, linting, and type checks.
- **Velocity** – Iterate quickly while preserving quality.
- **Ownership** – Treat every contribution as production-caliber work.

## Getting Started
1. Clone the repository and review task instructions.
2. Install dependencies and verify the build.
3. Implement the assigned challenge using the provided tooling.
4. Run tests, document results, and submit your work for review.

Build boldly.
